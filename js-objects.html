JavaScript Objects

A short subtitle

July 18, 2020

The simplest and most common way of creating javascript objects in the Object Literal. The syntax is as follows: 


           1 | let  car = {
           2 |  name: "bmw", 
           3 |   color: "white"
           4 | }
           5 | console.log(car.name)     // bmw
           6 | console.log(car.color)    // white
           7 | console.log(car[name])    // bmw
           8 | console.log(car[color])   // white
As shown above, access the properties of the object we use either the dot notation or the bracket notation. I'll explain these different choices later.
In javascript you can change the shape of the object after it is created. Let's add an model property to our car object. 

          car.model = 123;
          console.log(car.model)    // 123
We can also add functions to objects. 

          car.hoot = function () {
            console.log("Pipipiii")
          }
          car.hoot()        //Pipipiii
I can as well add the function as part of my bject literal and get the same result ike this: 

          1 | let  car = {
          2 |  name: "bmw", 
          3 |  color: "white",
          4 |  hoot: function() {console.log('Pipipiii')}
          4 | }
          5 | console.log(car.hoot)     // Pipipiii

The limit of creating objects using the Object literal method comes when we want to create multiple instances of the object. Let's see how a Constructor function works. 


          function Car() {
            this.name: 'bmw';
            this.color: 'white';
          }
          let new car = new Car();

          console.log(cat)     // Car {
                                    name: bmw
                                    color: white
                                  }
To understand what's really going on here we need to look closely at the this keyword. The this keyword refers to an object, but it does not refer to the function as an object itself, it's a special object called a context for the function. It's whatever object executing the current peace of code. By default it is the global object and in a web browser that is the window object. So when we execute the Car function, this was refering to a new empty object. This is made possible by the new keyword which creates a new empty javascript object, set the context of this to that new object and then calls the Car function. Read the article the this keyword for a detailed explanation.
