JavaScript Objects

July 18, 2020

There are three main ways of creating javascript objects: Object Literals, Constructor functions and ES6 Classes

Object Literals

The simplest and most common way of creating javascript objects is using Object Literals. The syntax is as follows: 


           1 | let  car = {
           2 |  name: "bmw", 
           3 |   color: "white"
           4 | }
           5 | console.log(car.name)     // bmw
           6 | console.log(car.color)    // white
           7 | console.log(car[name])    // bmw
           8 | console.log(car[color])   // white

As shown obove, you access the properties of the object using either the dot notation or the bracket notation.
In javascript you can change the shape of the object after it has been created. Let's add a model property to our car object. 


          car.model = 123;
          console.log(car.model)    // 123

We can also add functions to objects. 


          car.hoot = function () {
            console.log("Pipipiii")
          }
          car.hoot()        //Pipipiii

You can as well add the function as part of the object literal and get the same result like this: 


          1 | let  car = {
          2 |  name: "bmw", 
          3 |  color: "white",
          4 |  hoot: function() {console.log('Pipipiii')}
          5 | }
          6 | console.log(car.hoot)     // Pipipiii

Constructor functions

The limit of creating objects using the Object literal method comes when we want to create multiple instances of the object. A Constuctor function is used to instantiate a new object. I t looks like a regular function but it behaves diferently. Let's see how a Constuctor function solve this problem. 


          function Car() {
            this.name: 'bmw';
            this.color: 'white';
          }
          let new car = new Car();

          console.log(cat)     // Car {
                                    name: bmw
                                    color: white
                                  }

Notice that the function Car() starts with a capital letter. This is a convention for Constructor functions.

To understand what's really going on here we need to look closely at the this keyword. The this keyword refers to an object, but it does not refer to the function as an object itself, it's a special object called a context for the function. It's whatever object executing the current peace of code.

By default it is the global object and in a web browser that is the window object. So when we executed the Car function, this was refering to a new empty object.

This is made possible by the new keyword which creates a new empty javascript object, set the context of this to that new object and then calls the Car function. Read the article the this keyword for a detailed explanation.

Now let's make our object create any type of car instead of just one white bmw. 


          function Car(name, color) {
            this.name: name;
            this.color: color;
          }
          let new car = new Car('bmw', 'white');

          console.log(cat)     // Car {
                                    name: bmw
                                    color: white
                                  }

Classes

ES6 version of javascript provides functionality to create objects in a syntax similar to a statically typed language. This is just syntactic sugar like the two above of creating objects. I will later on explain a brute syntax of what will actually be happening under the hood when we create objects. All these methods will be doing the same thing underneath

To create our car using a class we will do the following. 


          class  Car {
            constructor(name, color) {
              this.name = name;
              this.color = color;
            }
            hoot() {
              console.log('Pipipiii')
            }
          }
          let car = new Car('bmw', 'white');
          car.hoot()             // Pipipiii
          console.log(car)       //  {
                                      name: bmw
                                      color: white
                                    }

Object.create

The methods above are basically syntactic sugar for Object.create. Let's create the same object using the Object.create syntax. 


          let car = Object.create(Object.prototype,
          {
            name: {
              value: 'bmw',
              enumerable: true,
              writable: true,
              configerable: true
            },
            color: {
              value: 'white',
              enumerable: true,
              writable: true,
              configerable: true
            }
          })
          console.log(car)         //  Object {
                                          name: bmw;
                                          color: white
                                        }

This is more involving and I think you can see why it's easier to use the methods explained above in creating objects. I expained the Object.create in more details in the article Javascript Object Properties.
