JavaScript Fuction Scope.

July 18, 2020

The let keyword has block scoping. The var keyword behaves differently and it is encouraged to always use the let keyword. You can read more about them here In JavaScript, variables are either in or out of scope. Consider the function below: 

        
        function learningCode(day) {
          let topic = "Functions are fun!!!";
        };
        learningCode(1);
        console.log(topic);    // Error

The console.log(topic) throws a reference error and not Functions are fun!!! because with the let keyword, topic is only found inside the learningCode function (i.e between the curly braces { }). The variable topic has function scope. With that concept in mind, consider the following block of code 

        
        function learningCode(day) {
          let topic = "Functions are fun!!!";
          let subTopic = function funScope() {
            console.log(topic)   // "Functions are fun!!!"
          }
          subTopic();
        };
        learningCode(1);

We have a function expression set to subTopic which calls the variable topic. Notice that we receive a valid output even though topic is not declaired inside the function expression. The reason for this behavior is that functions will look up the function chain (inside their parent functions) for variables called inside them but not declaired there. This is done until it finds the variable otherwise, it gives back a reference error.

Now let's declair a variable inside the function expression as shown here: 

        
        function learningCode(day) {
          let topic = "Functions are fun!!!";
          let subTopic = function funScope() {
            let topic = "replaced";
          }
          subTopic();
          console.log(topic) // "Functions are fun!!!";
        };
        learningCode(1);

When we execute subTopic, topic is set to "replaced". But when we log out topic outside the function expression subTopic, we get the message // "Functions are fun!!!". So we say the function expression's variablehas gone out of scope. The nested function can inherit it's parent's variables but the parent cannot.
